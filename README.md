## What Is This System?

MindSpark MusicWorks™ is a repeatable operating system for releasing gospel music. It takes one approved album or song project and turns it into 12 production-ready outputs — from theological review to distribution package to marketing campaign to ministry assets to revenue tracking — all with human approval at every gate.

This is not a content generator. It is a structured, auditable pipeline that keeps the Gospel at the center and the founder in control.

## How to Use This System

1. **Read the business brief first:**
   `/business_brief/mindspark_musicworks_business_brief.md`

2. **Fill out the intake form for your project:**
   `/inputs/album_intake_template.md`

3. **Follow the runbook:**
   `/runbooks/fire_and_flow_album_launch_runbook.md`

4. **Run each agent in order (1–13):**
   `/agents/`

5. **Review the sample output to see what the finished product looks like:**
   `/outputs/sample_fire_and_flow_album_launch_brief.md`

## Core Principle

**Nothing deploys automatically without human approval.**

The system may generate, organize, check, and recommend — but the founder must approve before any project goes public. Hard gates exist at Theology Review (Agent 03), Audio QC (Agent 04), and Final Approval (Agent 13). No amount of completed work bypasses a gate.

## Current Version

**V1 — OS Skeleton**
This version is a document-based system. All agents are instruction files. All workflows are runbooks. No live API connections. No automation. No deployment.

This is intentional. Prove the workflow manually first. Then automate what is working.
