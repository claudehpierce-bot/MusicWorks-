## What Is a Runbook?

A runbook is a step-by-step instruction manual for completing a repeatable process. This runbook describes the exact path from "we have an album idea" to "the album is live on streaming platforms with a full marketing and ministry package." Every future album or EP follows this same path. Nothing is skipped. Nothing is guessed.

The runbook references the 13 agents in order. Each step tells you which agent to run, what it needs, what it produces, and whether human approval is required before moving on.

---

## Pre-Conditions (Before You Start Step 1)

Before opening this runbook on a new project, confirm the following:

- [ ] The album concept has received informal founder approval ("Yes, we're making this")
- [ ] A project folder has been created in `/fire_and_flow_validation/[album_name]/`
- [ ] The Album Intake Template has been filled out (see `/inputs/album_intake_template.md`)
- [ ] A target release date has been estimated (can adjust later — but must have a starting target)
- [ ] Audio recordings are either complete, in progress, or scheduled (rough mixes acceptable at Step 1)

## STEP 1: Ingest Album Information
**Agent:** None (Human task)
**Time Required:** 1–2 hours
**Status Gate:** None

### What You Do
Fill out the Album Intake Template completely. Every field matters — the quality of every downstream agent output depends on the quality of what goes in here.

### What You Produce
- Completed `/inputs/[album_name]_intake.md`
- This becomes the source of truth for all agents on this project

### Checklist
- [ ] All required fields in the intake template are complete
- [ ] Track list is final (or best available at this stage)
- [ ] Lyrics are drafted (even rough drafts — they will be reviewed in Step 2)
- [ ] Bible themes are identified
- [ ] Target audience is described specifically
- [ ] Release date estimate is recorded

### Founder Note
Do not rush the intake form. A vague intake produces vague agent outputs. If you are unsure about a field, write your best current thinking — it can be refined later, but something is always better than nothing.

## STEP 2: Market Research & Gospel Positioning
**Agents:** 01 — Market Signal Researcher, 02 — Gospel Offer Architect
**Time Required:** 2–4 hours (agent run time)
**Status Gate:** Founder reviews both outputs before proceeding

### What Agent 01 Produces
- Market positioning brief
- Trending keywords and hashtags
- Competitive gap analysis
- Audience profile card

### What Agent 02 Produces
- Release concept brief (the one sentence)
- Offer structure (core + bonus + ministry)
- Spiritual hook statement
- Visual language recommendations

### Founder Review Required
Read both outputs. Ask yourself:
- Does this positioning feel true to what this album is?
- Does the spiritual hook statement represent the Gospel faithfully?

**Sign off:** Add a note to both files: "Reviewed and approved by [Name] on [Date]"

## STEP 3: Theological Review — HARD GATE
**Agent:** 03 — Theological Integrity Reviewer
**Time Required:** 1–3 days (includes human reviewer time)
**Status Gate:** HARD. Project does not move forward until Theology Approval Certificate is signed.

### What This Step Produces
- Track-by-track theology review
- Scripture mapping table
- Red flag list (if any)
- Suggested corrections (if needed)
- Signed Theology Approval Certificate from named human reviewer

### Process
1. Run the Theological Integrity Reviewer agent with all lyrics, titles, and marketing copy
2. Share the output with your named theology reviewer (pastor, elder, or trained theologian)
3. Address every red flag and every NEEDS REVISION item
4. Re-submit revised tracks for re-review
5. Obtain the signed certificate

### Hard Gate Rule
**Nothing proceeds past this point until the certificate is signed.** This is non-negotiable. The spiritual integrity of the release is the most important thing MindSpark MusicWorks™ protects.

---

## STEP 4: Audio, Artwork & File Quality Control — HARD GATE
**Agent:** 04 — Album Quality Control Agent
**Time Required:** 1–2 days (includes audio engineer time)
**Status Gate:** HARD. Distribution does not proceed until all tracks pass QC.

### What This Step Produces
- Audio QC checklist (pass/fail per track)
- Artwork QC checklist
- Preliminary metadata accuracy check
- Signed QC Certificate from audio engineer

### Process
1. Gather all master audio files, artwork files, and draft metadata
2. Run the Album Quality Control Agent checklist
3. For any FAIL items, return to the audio engineer or designer for correction
4. Re-check corrected items
5. Obtain the signed QC Certificate

### Hard Gate Rule
**Every track must pass. There are no exceptions for FAIL items.** Audio clipping or incorrect loudness on release will permanently damage the stream quality and cannot be recalled from most platforms.

## STEP 5: Build the Distribution Package
**Agent:** 05 — Metadata Distribution Agent
**Time Required:** 3–6 hours
**Status Gate:** Founder reviews before distribution is submitted

### What This Step Produces
- Master metadata spreadsheet (all tracks + album)
- Distribution platform checklist
- PRO and royalty registration checklist
- Distribution submission brief

### Prerequisites for This Step
- All ISRC codes registered
- UPC/EAN obtained
- All split sheets signed
- PRO registration complete
- SoundExchange registration complete

### Founder Review Required
Confirm: are all credits, names, and royalty registrations correct? This cannot easily be corrected after submission.

## STEP 6: Create the Marketing Package
**Agents:** 06 — SEO Content Agent, 07 — Short-Form Video Agent, 08 — YouTube Strategy Agent
**Time Required:** 4–8 hours (can be run in parallel)
**Status Gate:** Founder reviews all three outputs together

### What These Steps Produce
- YouTube descriptions, social captions, press release, SEO tags (Agent 06)
- Short-form video content calendar and 10+ concepts with scripts (Agent 07)
- YouTube channel optimization, thumbnail concepts, upload schedule (Agent 08)

### Running in Parallel
Agents 06, 07, and 08 can run at the same time — they do not depend on each other's outputs. This saves time.

### Founder Review Required
Review all marketing outputs together so the voice is consistent. Ask: does this all sound like Fire & Flow Gospel, or like three different people?

## STEP 7: Create the Ministry & Devotional Package
**Agent:** 11 — Devotional Asset Agent
**Time Required:** 4–6 hours
**Status Gate:** Founder reviews before distribution to ministry contacts

### What This Step Produces
- 7-Day Devotional Guide
- Small Group Discussion Guide
- Sermon Illustration Notes
- Prayer Cards
- Ministry Licensing Summary (placeholder)

### Note
This step can run in parallel with Step 6. Both Step 6 and Step 7 can run simultaneously after Step 5 is complete.

## STEP 8: Create the Playlist Outreach Plan
**Agent:** 09 — Playlist Outreach Agent
**Time Required:** 3–4 hours + ongoing outreach time
**Status Gate:** Founder reviews curator list before any pitch is sent

### What This Step Produces
- Tiered curator research list
- Personalized pitch templates per curator type
- Submission checklist
- Outreach tracker template

### Critical Timing Note
The Spotify editorial pitch must be submitted at least 7 days before the release date (4 weeks is strongly recommended). This step must be completed on time or the editorial pitch opportunity is lost for this release.

---

## STEP 9: Create the Email Campaign
**Agent:** 10 — Email Campaign Agent
**Time Required:** 3–5 hours
**Status Gate:** Founder reviews and approves voice before loading into email platform

### What This Step Produces
- 5–7 email sequence with subjects, preview text, body copy, and CTAs
- Segmentation notes
- Deliverability checklist

### Setup Required
Load the approved email sequence into the email platform (Mailchimp, ConvertKit, Flodesk, etc.) at least 7–10 days before the first send date.

---

## STEP 10: Create the Revenue Tracking Plan
**Agent:** 12 — Revenue Analytics Agent
**Time Required:** 2–3 hours
**Status Gate:** Founder reviews before launch

### What This Step Produces
- Revenue stream activation checklist
- 90-day tracking plan
- KPI dashboard template
- Revenue projection worksheet (conservative)
- Monthly reporting template

### Prerequisite Actions
Confirm all revenue streams are set up and active: distributor, PRO, SoundExchange, ContentID, direct sales platforms.

---

## STEP 11: Founder Approval Dashboard
**Agent:** 13 — Founder Approval Agent
**Time Required:** 2–4 hours (review and sign-off)
**Status Gate:** FINAL GATE. Nothing is released without this.

### What This Step Produces
- Complete launch readiness dashboard
- Open questions list (prioritized)
- Hard gate confirmation checklist
- Founder's Declaration (to be signed)
- Launch day checklist

### Process
1. Compile all agent outputs into a single project review folder
2. Run the Founder Approval Agent — it checks every output
3. Review the dashboard
4. Resolve all RED items (may require returning to earlier steps)
5. Acknowledge all YELLOW items in writing
6. Confirm all Hard Gate items are complete
7. Sign the Founder's Declaration

**Once signed, the project is officially launch-ready.**

---

## STEP 12: Launch Day Execution
**Agent:** None (Human task)
**Time Required:** Full day
**Status Gate:** None — this is the finish line

### What You Do
Follow the Launch Day Checklist produced by Agent 13. Typical launch day actions include:

**Morning:**
- Confirm all files are live on distribution platforms
- Verify Spotify, Apple Music, YouTube links are working
- Send launch day email (Email #3 from the campaign)
- Post launch social media content (from Agent 07 calendar)
- Upload YouTube video(s) (from Agent 08 schedule)

**Midday:**
- Monitor first streaming data (Spotify for Artists)
- Monitor YouTube views and engagement
- Respond to the first 20–30 comments on every platform
- Share the release in relevant community channels

**Evening:**
- Post an additional piece of short-form content (from calendar)
- Send any personal outreach to ministry contacts
- Review day-1 data and note anything surprising

**After Launch:**
- Begin 90-day tracking cadence (from Agent 12)
- Continue playlist outreach (from Agent 09 tracker)
- Continue email sequence (Emails 4–7)
- Keep posting short-form content (from Agent 07 calendar)
